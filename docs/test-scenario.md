# 수동 테스트 시나리오

> 프론트엔드: http://localhost:3001
> Swagger: http://localhost:3000/api/docs

---

## 시나리오 1: 회원가입 → 로그인

### 1-1. 회원가입
1. http://localhost:3001/signup 접속
2. 이름: `테스트유저`, 이메일: `test@test.com`, 비밀번호: `test1234` 입력
3. 회원가입 버튼 클릭
4. **기대**: 로그인 페이지로 이동

### 1-2. 이메일 중복 회원가입
1. 다시 /signup 접속
2. 동일 이메일 `test@test.com`으로 가입 시도
3. **기대**: "이미 등록된 이메일입니다." 에러 표시

### 1-3. 로그인
1. http://localhost:3001/login 접속
2. 이메일: `test@test.com`, 비밀번호: `test1234`
3. **기대**: 메인 페이지로 이동, 우측 상단에 "테스트유저님" 표시

### 1-4. 잘못된 비밀번호 로그인
1. /login에서 비밀번호를 `wrongpw`로 입력
2. **기대**: "이메일 또는 비밀번호가 올바르지 않습니다." 에러 표시

---

## 시나리오 2: 영화 목록 → 상세 → 상영 시간표

### 2-1. 영화 목록 조회
1. http://localhost:3001 접속
2. **기대**: 영화 7편이 카드 형태로 표시
3. 각 카드에 제목, 설명, 장르, 상영시간, 등급 확인

### 2-2. 영화 상세 조회
1. 아무 영화 카드 클릭 (예: 인터스텔라)
2. **기대**: 영화 상세 정보 + 날짜별 상영 시간표 표시
3. 상영 시간표에 시간, 상영관명, 가격 확인

---

## 시나리오 3: 좌석 선택 → 예매

### 3-1. 좌석 현황 조회
1. 상영 시간표에서 아무 시간 클릭
2. **기대**: 좌석 그리드 표시 (행 A~H or A~F or A~D, 좌석 1~10)
3. 모든 좌석이 "선택 가능" 상태 (회색)

### 3-2. 좌석 선택
1. 원하는 좌석 클릭 (예: A3)
2. **기대**: 해당 좌석이 검은색으로 변경 (선택됨)
3. 하단 버튼이 "A3 예매하기"로 변경
4. 다른 좌석 클릭하면 선택이 이동하는지 확인

### 3-3. 미로그인 상태에서 예매 시도
1. 로그아웃 상태에서 좌석 선택 후 예매 버튼 클릭
2. **기대**: 로그인 페이지로 이동

### 3-4. 로그인 후 예매
1. 로그인 후 다시 좌석 선택 페이지로 이동
2. 좌석 선택 → 예매하기 클릭
3. **기대**: "내 예매" 페이지로 이동, 방금 예매한 내역 표시

---

## 시나리오 4: 예매 내역 조회 → 취소

### 4-1. 예매 내역 확인
1. 상단 네비게이션 "내 예매" 클릭
2. **기대**: 예매 내역에 영화 제목, 날짜/시간, 상영관, 좌석번호, 가격, "예매완료" 상태 표시

### 4-2. 예매 취소
1. "예매 취소" 버튼 클릭
2. 확인 팝업에서 "확인" 클릭
3. **기대**: 상태가 "취소됨"으로 변경, 취소 버튼 사라짐

### 4-3. 취소한 좌석 재예매 확인
1. 같은 상영의 좌석 선택 페이지로 다시 이동
2. **기대**: 방금 취소한 좌석이 다시 "선택 가능" 상태

---

## 시나리오 5: 중복 예매 방지

### 5-1. 동일 좌석 중복 예매
1. 좌석 A1을 예매 완료
2. 같은 상영의 좌석 선택 페이지 재접속
3. **기대**: A1 좌석이 "예매됨" 상태 (회색, 클릭 불가)

### 5-2. Swagger로 직접 중복 예매 시도
1. http://localhost:3000/api/docs 접속
2. POST /api/reservations에서 이미 예매된 screeningId + seatId로 요청
3. **기대**: 409 Conflict, "이미 예매된 좌석입니다." 응답

---

## 시나리오 6: Swagger API 테스트

### 6-1. Swagger UI 접속
1. http://localhost:3000/api/docs 접속
2. **기대**: 인증, 영화, 상영, 예매 4개 태그로 분류된 API 문서

### 6-2. 회원가입 → 로그인 → 토큰 획득
1. POST /api/auth/signup 실행 (새 이메일)
2. POST /api/auth/login 실행
3. 응답의 accessToken 복사
4. 우측 상단 "Authorize" 버튼 → Bearer 토큰 입력

### 6-3. 인증 필요 API 테스트
1. 토큰 입력 후 GET /api/reservations 실행
2. **기대**: 200 OK, 빈 배열 또는 예매 목록

### 6-4. 토큰 없이 인증 API 접근
1. Authorize에서 토큰 제거
2. POST /api/reservations 실행
3. **기대**: 401 Unauthorized

---

## 시나리오 7: 엣지 케이스

### 7-1. 존재하지 않는 영화 조회
1. Swagger에서 GET /api/movies/00000000-0000-0000-0000-000000000000
2. **기대**: 404 Not Found

### 7-2. 잘못된 DTO 전송
1. POST /api/auth/signup에서 email 필드 없이 전송
2. **기대**: 400 Bad Request, validation 에러 메시지

### 7-3. 유효하지 않은 JWT 토큰
1. Authorize에 `Bearer invalid-token-here` 입력
2. GET /api/reservations 실행
3. **기대**: 401 Unauthorized
